


# 老奶奶的懷舊時光 專題展示講稿

## 一、專題介紹 (2-3分鐘)

### 專題概述

「老奶奶的懷舊時光 是一個完整的飲料店線上訂購系統，讓顧客能方便地在線上瀏覽菜單、下單及追蹤訂單進度。同時也為店家提供訂單管理、商品管理及會員管理等功能。」

### 專題特色

- 完整的購物系統：從選擇商品到完成結帳的流暢體驗
- 多角色權限管理：顧客、員工、管理員各有不同權限
- 訂單狀態追蹤：即時查看訂單處理進度
- 優惠券系統：提升顧客購買慾望
- 品牌故事專區：增強使用者體驗與品牌連結

## 二、系統展示 (8-10分鐘)

### 1. 管理端功能（先展示最高權限功能）

### 優惠券管理（Administrator 專屬）

**操作步驟**：「首先我要展示系統中權限等級最高的功能 - 優惠券管理系統。這個功能只有具有 Administrator 角色的用戶才能訪問和操作。讓我登入管理員帳號，進入優惠券管理頁面。」

「在這個頁面中，最高權限管理員可以創建、編輯和刪除優惠券，設置使用條件、有效期和折扣金額。讓我們新增一個優惠券看看整個流程。」

**技術說明**：「優惠券管理功能是通過 [Authorize(Roles = 'Administrator')] 特性進行保護的，確保只有最高權限的管理員才能管理這個敏感的促銷工具。這種角色控制是通過 ASP.NET Core Identity 框架實現的，同時在控制器層和視圖層都進行了權限檢查。」

「系統還實現了優惠券的使用記錄追蹤和統計分析功能，幫助管理員評估促銷效果。我們可以看到每個優惠券的使用次數和產生的折扣總額，這些數據對營銷決策非常有價值。」

### 權限管理展示

**操作步驟**：「現在讓我登出管理員帳號，並用普通員工帳號登入，然後嘗試訪問剛才的優惠券管理頁面。您可以看到系統直接拒絕訪問並顯示權限不足的提示，這證明了我們的權限控制系統運作正常。」

**技術說明**：「系統採用 ASP.NET Core Identity 框架實現了完整的身分驗證與授權機制。通過 [Authorize] 特性和基於角色的存取控制，確保只有授權用戶才能訪問對應功能。系統設定了不同的角色層級，如 Customer、Employee 和 Administrator，每個角色擁有不同的權限範圍。在數據查詢時，系統還會通過 ClaimsIdentity 獲取當前用戶 ID 並加入過濾條件，實現了多層次的安全防護。」

### 訂單管理系統

**操作步驟**：「使用員工帳號，我們可以訪問訂單管理界面。這裡可以查看所有訂單，支持按狀態、日期等條件篩選，並能直接變更訂單狀態。」

**技術說明**：「訂單管理界面使用了 DataTables 插件實現分頁、排序和搜尋功能，大幅提升了管理效率。後端通過 Web API 回傳 JSON 數據，採用 AJAX 非同步載入，減輕了伺服器負擔。狀態管理使用 Repository Pattern 中的專用方法處理業務邏輯，遵循單一責任原則。」

**操作步驟**：「當點擊『接受訂單』按鈕時，訂單狀態會更新為『處理中』，同時在下方的追蹤記錄中會自動添加一條新記錄。」

**技術說明**：「每次點擊狀態按鈕都會觸發 OrderController 的對應 Action 方法，該方法通過 UnitOfWork 調用 OrderHeaderRepository 更新訂單狀態，同時創建新的 OrderTracking 記錄。」

### 後台商品管理

**操作步驟**：「在管理後台，我們可以輕鬆進行商品的添加、編輯和刪除操作。這裡展示的是商品列表，支持搜索和分頁功能。」

**技術說明**：「後台管理採用 ASP.NET Core MVC 框架，結合 Entity Framework Core 實現 CRUD 操作。我實作了完整的增刪改查功能，並使用 Entity Framework Core 的遷移功能管理資料庫結構變更。使用 ViewModels 分離展示邏輯與資料邏輯，增強了代碼維護性。」

### 2. 顧客端功能

### 首頁與商品瀏覽功能

**操作步驟**：「現在讓我們登出管理帳號，使用一般顧客身份瀏覽網站。這是我們的網站首頁，展示了所有的商品分類和熱門商品。」

**技術說明**：「首頁採用 Bootstrap 5 的響應式網格系統，確保在各種裝置上都有最佳顯示效果。商品資料通過 Entity Framework Core 從資料庫讀取，我使用了 Repository Pattern 來隔離數據訪問邏輯，使代碼更易於維護和測試。」

### 購物車功能

**操作步驟**：「現在我們將幾個商品加入購物車。注意看，每次添加商品時，頂部導航欄的購物車圖標旁邊的數字會即時更新。」

**技術說明**：「購物車功能使用了 jQuery 與 AJAX 實現無刷新更新，大幅提升了用戶體驗。後端通過 Session 管理購物車狀態，同時結合 Identity Framework 將購物車數據與當前用戶關聯。」

### 收藏功能

**操作步驟**：「注意看每個商品卡片右上角的愛心圖標，這是我們的商品收藏功能。當我點擊這個愛心時，系統會顯示確認收藏的提示，同時導航欄的收藏數量會更新。」

**技術說明**：「收藏功能採用了 AJAX 技術實現無刷新交互，提示效果使用了 SweetAlert2 庫。我還實現了前端防抖處理，即使用戶連續多次點擊，也只會發送一次請求，優化了性能。」

### 結帳流程

**操作步驟**：「讓我們進入結帳頁面，填寫訂單信息並套用優惠券。系統會即時計算優惠後的金額。」

**技術說明**：「結帳流程整合了 ASP.NET Core 的 Model Validation 進行表單驗證。優惠券功能通過 AJAX 即時查詢並應用折扣。後端利用 Entity Framework Core 的事務處理確保訂單數據的完整性和一致性。」

### 訂單追蹤功能

**操作步驟**：「在『我的訂單』頁面，用戶可以查看訂單狀態和詳細進度。這個水平進度條直觀顯示了訂單處理的當前階段，下方的時間軸則展示了完整的狀態變更歷史。」

**技術說明**：「訂單追蹤功能的核心是 OrderTracking 實體與 OrderHeader 的一對多關聯設計，記錄了完整的訂單狀態變更歷史。進度條使用了 Bootstrap 組件和自定義 CSS，配合 Razor 語法中的條件判斷動態添加高亮效果。」

### 3. 品牌故事專區功能

**操作步驟**：「讓我們瀏覽『老奶奶的故事』專區，這裡收集了品牌的核心故事和飲品起源。每一個故事都以懷舊風格呈現，讓顧客能感受到品牌的歷史和文化底蘊。」

**技術說明**：「故事專區使用了 Entity Framework Core 與 Repository Pattern 實現數據訪問。我們創建了 Story 模型來儲存不同類型的故事內容，並使用 StoryRepository 來處理特定查詢需求。前端使用 Bootstrap 5 的卡片組件和自定義 CSS 實現了具有復古風格的視覺效果。」

## 三、技術亮點 (3-5分鐘)

### 後端技術

- ASP.NET Core 8 Razor Pages：最新的微軟網頁開發框架
- Entity Framework Core：使用 Code First 建立強型別資料庫結構
- Repository Pattern 與 Unit of Work Pattern：強化程式碼的可維護性與可測試性
- 依賴注入 (DI)：鬆耦合的系統架構設計

### 前端技術

- Bootstrap 5：響應式前端框架，確保網站在各種裝置上有良好顯示
- jQuery 與 JavaScript：增強使用者體驗的互動功能
- DataTables：強大的表格處理與資料呈現
- SweetAlert2 與 Toastr：美觀的通知與提示功能

### 資料庫設計

- SQL Server：使用關聯式資料庫儲存應用資料
- Migration 機制：便於資料庫版本控制
- 外鍵關聯：維持資料完整性

### 安全性實作

- ASP.NET Core Identity：使用者認證與授權系統
- 基於角色的存取控制 (RBAC)：不同角色具有不同操作權限
- 防範 CSRF 攻擊：使用 Anti-forgery Token

## 四、實作難點與解決方案 (1-2分鐘)

### 多角色權限管理挑戰

- 設計了細緻的權限系統，通過角色基礎的授權機制
- 使用 [Authorize] 特性確保權限控制在控制器層級

### 訂單狀態流程設計

- 實作狀態機制，確保訂單處理流程的準確性
- 使用 Repository 模式集中管理訂單狀態變更邏輯

### 前端資料處理效能

- 使用 AJAX 與 DataTables 實現高效的資料載入
- 改善 JSON 序列化處理避免循環參考問題
